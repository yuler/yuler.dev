# Thoughts 索引页：便利贴 + 无限画布 — 设计说明

**状态：** 已评审（与产品方对话确认）  
**日期：** 2026-05-01  
**范围：** `/thoughts` 列表页的展示与交互；不新增单篇 thought 独立路由（当前仓库仅有 `src/pages/thoughts/index.astro`）。

## 1. 目标

- 索引页呈现 **便利贴（sticky note）** 视觉：色纸、阴影、层次，可选轻度纸质/边框其一，避免堆叠过多装饰。
- 在合适环境下提供 **可平移、可缩放的画布**（逻辑上的「无限」：按内容包围盒 + 留白扩展可探索区域，而非数学意义上的无限平面）。
- **小屏** 与 **`prefers-reduced-motion`** 有明确降级路径，避免强依赖手势或动效才能完成阅读。

## 2. 已锁定决策

| 主题 | 选择 | 说明 |
|------|------|------|
| 笔记在画布上的分布 | **C：时间分区** | 按 `date` 分带（如按年或「最近一批」）；每带占据画布上大致区域（例如较新的偏右上）；带内位置仍 **确定性**（由 slug/id 哈希派生 x、y、小角度旋转），**同 URL 刷新位置不变**。 |
| 小屏 | **C：混合** | 默认 **可滚动列表式便签**；提供 **「展开画布」** 进入与桌面类似的平移/缩放视图。 |
| `prefers-reduced-motion: reduce` | **C：仅列表** | 全 viewport **只使用列表布局**；**不展示「展开画布」入口**；便签保留配色与阴影等静态层次；旋转为 0 或极小固定角。 |

## 3. 体验细则

### 3.1 桌面 / 宽屏（且非 reduced-motion）

- 默认：**画布模式**。
- 平移：指针拖拽（必要时配合滚轮平移策略，避免与浏览器纵向滚动冲突——画布区域应 `overscroll-behavior` 或等价处理）。
- 缩放：滚轮或触控板缩放，设 **最小/最大缩放**，防止内容消失或过大。
- 便签在画布内 **可被 Tab 聚焦**；建议 **焦点顺序按时间（新→旧或旧→新，择一并在实现中固定）**，与屏幕空间位置解耦，降低认知负担。

### 3.2 小屏（常规动效偏好）

- 默认：**列表模式**（便签外观的纵向流式布局，可多列若宽度允许）。
- **「展开画布」**：进入画布模式；入口文案需清晰（例如「在画布上查看」），退出方式明确（关闭/返回列表）。

### 3.3 Reduced motion

- **仅列表**；**无画布入口**。
- 无飘动、视差、惯性动画；旋转按上表弱化。

### 3.4 内容与媒体

- 继续使用现有 **Markdown 渲染与 `Thought` 内容结构**（日期、标签、正文、可选图片）。
- **LightBox** 与现有一致；便签内图片的 `data-lightbox` 等行为保持不变。

### 3.5 链接与锚点

- 当前无单篇 thought 页面：便签 **不必** 整块包裹外链；若需分享定位，可为每条便签容器设置稳定 `id`（如来自 slug），便于将来扩展锚点或独立路由。

## 4. 技术方向（实现约束）

- **推荐方案：** Astro 页面 + **小型 client island**（`client:load` 或等价），用 CSS `transform` 实现画布平移与缩放；避免为重交互单独引入大型 UI 框架，除非实现中证明手势成本过高。
- **不推荐：** WebGL/纯 Canvas 重绘正文（与可选择文本、无障碍、Lightbox 集成成本高）。

### 4.1 建议文件职责

- `src/pages/thoughts/index.astro`：数据获取、面包屑、页头；根据 **媒体查询 + `prefers-reduced-motion`** 决定渲染列表壳 + 是否挂载画布 island 与「展开画布」入口。
- 新建画布相关组件（名称可调整）：例如 `ThoughtsCanvas.astro`（island）负责变换、指针/滚轮、子元素绝对定位。
- 便签外观：新建 `ThoughtSticky.astro` 或为 `Thought.astro` 增加 `variant` / wrapper，**共用同一套 `render(thought)` 输出**，避免两套 Markdown 管道。

### 4.2 位置算法（摘要）

- 输入：有序 thoughts（已与现网一致按日期排序）、每条 slug、日期。
- 步骤：按日期分带 → 为每带定义画布上的目标矩形区域 → 在矩形内用 **哈希确定性** 分布 (x, y, rotate) → 根据所有便签计算包围盒，扩展为可平移的世界尺寸。

## 5. 非目标（YAGNI）

- 用户自定义拖拽便签并持久化位置。
- 多用户协作、实时同步画布。
- 单篇 thought 独立 URL（除非另开需求）。

## 6. 验收要点（实现后自测）

- 宽屏：画布可平移、缩放受限合理；便签位置刷新不变。
- 小屏：默认列表可读；「展开画布」可用且可退回列表。
- 系统开启「减少动态效果」：仅列表、无画布入口；焦点与阅读正常。
- 含图片的 thought：LightBox 仍可用。
- 键盘：列表与画布模式下焦点路径可理解、无焦点陷阱。
